# Kafka Message Streaming System

A simple Kafka-based message streaming system with Reddit comment processing.

## Architecture

- **reddit_streamer**: Producer that sends Reddit comments from CSV to Kafka
- **comment_processor**: Consumer that processes messages (1s delay) and logs timestamps
- **metrics_aggregator**: Generates throughput (Mbps) and latency (ms) reports

## Quick Start with Docker

### 1. Build and start the system
```bash
docker-compose up --build -d
```

This will start:
- Single Kafka broker (KRaft mode)
- Kafka UI at http://localhost:8080
- 1 producer instance
- 2 consumer instances (same group, load balanced)

### 2. Wait for Kafka to be ready (~30 seconds)
```bash
# Check if Kafka is healthy
docker-compose ps

# Wait for kafka to show "healthy"
```

### 3. View logs
```bash
# Producer logs
docker-compose logs -f producer

# Consumer logs
docker-compose logs -f consumer1 consumer2

# All logs
docker-compose logs -f
```

### 4. Generate metrics report
```bash
docker-compose run --rm aggregator
```

Reports are saved to Docker volume. To access:
```bash
# Copy report to current directory
docker run --rm -v message-streaming-system_reports:/reports -v $(pwd):/output alpine cp -r /reports /output/
```

### 5. Stop the system
```bash
docker-compose down
```

To remove volumes (clears Kafka data and logs):
```bash
docker-compose down -v
```

## Monitoring Progress

### Watch messages being processed
```bash

docker-compose logs -f consumer1 consumer2
```

### Check Kafka UI
Open http://localhost:8080 and:
- Click "Topics" → "reddit-comments"
- View messages, partitions, consumer groups
- Monitor lag and throughput

### View CSV logs directly
```bash
# Access consumer container
docker-compose exec consumer1 cat /app/logs/processing_logs_reddit-processors.csv
```

## Troubleshooting

### Services fail to start
```bash
# Clean up and restart
docker-compose down -v
docker-compose up --build -d
```

### Check Kafka health
```bash
docker-compose ps kafka
# Should show "healthy" status
```

### View all service logs
```bash
docker-compose logs
```

### Producer finishes before consumers start
This is normal - consumers will process all messages from the topic.

## Advanced Usage

### Run with different dataset
Edit `docker-compose.yaml`:
```yaml
producer:
  environment:
    DATASET_PATH: datasets/your_dataset.csv
```

### Adjust processing speed
```yaml
consumer1:
  environment:
    PROCESSING_DELAY_SEC: 0.5  # Faster processing
```

### Add more consumers
```bash
docker-compose up -d --scale consumer1=3
```

## Local Development (without Docker)

### Setup
```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

### Start Kafka only
```bash
# Use the simplified single-broker setup
docker-compose up -d kafka
```

### Run services locally
```bash
# Update .env file
KAFKA_BOOTSTRAP_SERVERS=localhost:9092

# Run services
python -m reddit_streamer.main
python -m comment_processor.main
python -m metrics_aggregator.main
```

## Metrics Explained

Generated by the aggregator:

- **Throughput (Mbps)**: Data transfer rate in megabits per second
- **Latency (ms)**: 
  - Max: Worst-case processing time from send to completion
  - Avg: Average processing time
  - Min: Best-case processing time
- **Messages/sec**: Processing rate
- **Consumer counts**: Messages processed per consumer
